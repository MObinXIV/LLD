# Factory (Virtual constructor)

## What's factory 

It's as the name said a factory which combine all the similar guys in one factory.


- Creating objects of similar type.
- hide the complexity of object creation.
- Client code has no idea about object  creation.


## Why to use it 

to keep the user away from the unnecessary details, say user wanna to purchase a course, we want him to ask only for the course.

## How 

We get all the similar kinds of one abstract thing and combine them into one interface.

### Pros 

1. Guarantee abstraction.
2. code flexible & adaptable
3. Very useful for frameworks & libraries.

### Cons

1. Complex code. 
2. Takes time to set the base.
3. Not a pattern that can be refactored into.
